# AGENTS.md

## Commit Messages

Use Conventional Commits when possible:

```text
<type>: <summary>
```

Keep the summary short and focused on the main change.

## Types

- `feat`: new post, new page, or new site feature
- `docs`: edits to existing writing, links, wording, or README
- `fix`: bug fix, broken link/image, broken build, or factual correction

## Quick Rules

- New post: use `feat`
- Edit existing post: use `docs`
- Correct an actual problem: use `fix`

## Notes

- Do not mention routine `updated` timestamp changes in the summary unless that is the main point
- Prefer the reader-facing change over internal details

## Examples

- `feat: publish bus tech stack post`
- `docs: refine links and wording in bus tech stack post`
- `fix: correct map tile licensing note in bus tech stack post`
