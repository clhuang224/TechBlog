忽然想分享一下我一直有的工作習慣。  
其實應該每個人多多少少都會在工作中做筆記，但我是習慣每天做紀錄。

讀語聽系的時候，為了個案權益及自身的學習，多半都需要寫課節計畫、治療紀錄、評估報告或進展報告等。  
這些記錄一定包含可量化的目標、數據跟語言樣本，可以用來比較前後測的結果，也能回顧治療涵蓋的向度。  

<!-- more -->

開始工程師的工作之後，我也帶著這樣的思考方式記錄每天接觸的項目跟相關的枝微末節。

方式很簡單，就是開一個 Notion 的 Database，記下每天的內容。  
大概長這樣：

```md
- Name（文字）：`20260204 We`
- Date（日期）：`2026/2/4`
- Event（多選）：`站會`、`讀書會`、`生日會`

- 昨天
    - task A：上測試環境
    - task B：正在處理ＯＯＯ
- 今天
    - task B：繼續處理ＯＯＯ

## 站會
A 同事：balabala
B 同事：balabala
C 同事：請假
D 同事：都在開會沒寫 code

## 生日會
紅葉蛋糕好吃
```

包含站會要說的話，和當天任何記錄下來的東西。
日誌的欄位包含篇名、日期、會議標籤，這樣之後如果想要回憶某個會議的內容，就很好找。

我作筆記的方式有時是類似打逐字稿，因為我覺得在聽打的當下要把重點抽出來，會消耗一部分的認知資源、拖慢速度，之後回頭看紀錄，會看不到全貌。
所以我有時是聽到什麼就直接原封不動打字，可能包含一些聊天的廢話，或是一些情境的描述，如果有聽不懂的詞就先空耳。  
站會聽別人的內容，也會盡可能全部記下來。

以神經的角度來看，逐字稿搭配情境描述的回憶效果肯定比只寫關鍵字好，因為會連結到更多的感官記憶。  
不過這可能跟每個人接受資訊的路徑差異有關。  
我自己在聽理解的速度不是那麼快，所以利用語音轉文字的方式，用視覺、閱讀理解的能力來掌握語意。  
但有些人可能在筆記的當下就已經做完整理了，這就是每個人的優勢不同。

做日誌聽起來沒什麼，但長期下來，當專案管理平台撈不到某個久遠的決策時，就有可能從自己的筆記裡挖到相關的紀錄，平常也有利於寫週報、月報、季度回饋等。  
技術方面的關鍵字，雖然零碎，在未來某一天也可能觸發新的靈感。
另外聽會議做筆記比較不會睡著。

工作日誌對於工作效率可能沒有明顯的影響，但對於了解專案和決策的整體脈絡，及自我成長，對我來說是不可或缺的。
